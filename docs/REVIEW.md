# Ревью изменений (Diff между старой логикой и новыми методами)

В данном ревью собраны изменения, вносимые в архитектуру приложения согласно плану, для перевода проекта в изоляцию, добавления LITE-клиента, а также обновления скриптов развертывания.

## 1. Cloudflare Worker (EDGE INGRESS)

**Было:**

- `cloudflare-worker.ts` был простой прокси-оберткой. Он безусловно брал запрос и перенаправлял его на исходный `ORIGIN_URL`.
- Отсутствовала встроенная обработка CORS-preflight (`OPTIONS`), клиенты могли сталкиваться с блокировками.
- Проксировал всё без разбора политик CORS и возможностей сокрытия ключей.

**Стало:**

- Добавлена полноценная генерация правильных заголовков `CORS` и маршрут для быстрого ответа на preflight-запросы (метод `OPTIONS`).
- Разрешается прямой доступ (Origin) только для доверенных расширений `moz-extension://`, `chrome-extension://` или конкретных доменов через `ALLOWED_ORIGINS`.
- Реализована безопасная инъекция ключей LLM (на примере `GEMINI_API_KEY`) прямо на уровне Edge (маршрут начинает обрабатываться через `/api/gemini`), что позволяет отправлять запросы напрямую в Google минуя свой Backend, без необходимости разглашать API-ключ LITE-клиенту. Узел Edge берет на себя роль безопасного Ingress.

## 2. Адаптация UI для LITE (Selector Endpoints)

**Было:**

- При выборе источника "Gateway / Cloud Engine" в сайдбаре пользователю отображалось одно пустое текстовое поле для кастомного URL.
- Пользователю приходилось руками писать порты `http://localhost:3000` (Node) или `http://localhost:11434` (Ollama), если он тестировал систему локально. Это не интуитивно.

**Стало:**

- Переписан компонент `web/src/ui/SettingsSidebar.tsx`.
- Введен удобный выпадающий список для Endpoint, который предлагает заранее настроенные опции "в один клик":
  1. `Локальный Gateway (:3000)`
  2. `Напрямую в Ollama (:11434)`
  3. `Cloudflare Edge / Custom` (автоматически раскрывает текстовое поле для кастомного адреса).
- При переключении URL прозрачно пробрасывается в `pingUrl` стейта, не ломая старый API-клиент.

## 3. Разделение слоев Docker-Compose (Fix "The Zoo")

**Было:**

- Запускалось лишь 2 контейнера (`gateway` и `ocr`).
- Образ для `gateway` собирался из файла `nginx.Dockerfile` (в котором смешивался Node.js и Nginx). А сам веб-сервер Nginx в файле настроек `nginx.conf` перекидывал API `/api/` и `/convert` сразу в _контейнер OCR Python_ (`http://ocr:8000`), вообще минуя логику Node.js Gateway.

**Стало:**

- Четкое расслоение на 3 изолированных контейнера (`docker-compose.yml`), каждый несет свою зону ответственности:
  1. **Nginx** (порт `80` внутри контейнера): строится из `docker/nginx.Dockerfile`. Раздает только сгенеренную Vite статику React и выступает балансировщиком-прокси. Перенаправляет `/api/` в Gateway.
  2. **Gateway** (внутренний порт `3000`): строится из `docker/gateway.Dockerfile` на `node:22-slim`. Теперь Nginx перекидывает все запросы в этот контейнер, и Node.js шлюз принимает решения.
  3. **OCR Engine** (внутренний порт `8000`): Мощная изолированная машина на базе Python. Порт торчит только внутрь Docker сети.

## 4. Скрипты развертывания и тестирования (Bun vs Node)

**Было:**

- Существовал единый `run.sh` — огромный баш-скрипт запускающий и Node и Python в фоне. Не было изоляции скриптов.
- Скрипт отладки `scripts/debug.sh` тестировал JS часть локально через стандартный `npm test`. Он ничего не знал про `Bun`, хотя движок заявлен как мульти-рантаймовый. Краш в Bun выяснялся только в пулл-реквестах GitHub Actions.

**Стало:**

- Старый монолит `run.sh` удален. Созданы инкрементальные и узконаправленные Bash скрипты:
  - `scripts/run-docker.sh` — Поднимает `docker-compose`. Максимально изолированная продакшен-среда.
  - `scripts/run-local.sh` — Поднимает VENV Python с UVICORN и Node.js/Bun. Идеально для быстрой разработки без ребилдов докера.
  - `scripts/build-lite.sh` — Собирает чистую статику LITE-клиента.
- В `scripts/debug.sh` встроена локальная матрица тестов. При наличии интерпретатора Bun, скрипт автоматически запускает `bun test` вслед за `npm test` Node.js. Любая несовместимость синтаксиса в Bun теперь бьет разработчика по рукам еще до заливки в репозиторий.
