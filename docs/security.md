# Политика безопасности

Архитектура системы разработана с учетом строгой изоляции данных конфиденциальности.

- **Изоляция сетевого слоя (Docker)**
  - Во внешнюю среду опубликован только шлюз Nginx.
  - API Gateway и внутренний сервис OCR работают в изолированной виртуальной сети и недоступны извне напрямую по открытым портам.

- **Обработка данных (Zero-Retention)**
  - Приложению не требуется база данных.
  - Файлы, обрабатываемые на стороне сервера, помещаются во временный файл через Python `tempfile` и удаляются после распознавания.
  - Локальный API Gateway не логирует и не сохраняет исходные документы пользователя.

- **Режим локальной обработки (Браузер)**
  - При выборе "Browser Engine" исходный образ не покидает вкладку браузера, вся обработка происходит за счет WebAssembly-порта Tesseract в оперативной памяти устройства.

- **Ключи внешних API (LLM Cloud API)**
  - Ключи доступа к сторонним LLM (например, Gemini или OpenRouter) запрашиваются у пользователя на стороне Frontend.
  - Они не сохраняются в системе, не кешируются внутренним Gateway, и передаются в сторонние платформы напрямую из браузера приложения.
  - Механизм хранения полностью опирается на возможности менеджера паролей браузера пользователя.
