# Ветки развития и заглушки

[Документация](../README.md) | [История roadmap](./history.md)

Снимок проверен 19 июня 2026 года. Этот файл описывает фактически созданные
локальные ветки, чтобы пустое имя ветки не принимали за готовую реализацию и
не использовали как источник случайного cherry-pick.

## Заглушки

| Ветка                      | Фактическое состояние                                                    | Предполагаемое направление                | Как использовать                                                                     |
| -------------------------- | ------------------------------------------------------------------------ | ----------------------------------------- | ------------------------------------------------------------------------------------ |
| `extension`                | указывает на тот же commit, что `main`; уникальных изменений нет         | browser extension                         | начинать новую реализацию от актуальной базы, не считать ветку готовой feature       |
| `engine-time-optimization` | совпадает с checkpoint `engine-merged-pr5`; уникальных perf-коммитов нет | измерение и сокращение времени OCR        | перед работой синхронизировать с нужной базой и добавлять только измеримый perf diff |
| `engine-tesseract-quality` | совпадает с тем же checkpoint; уникальных quality-коммитов нет           | отдельные эксперименты качества Tesseract | использовать для изолированного quality diff с oracle и порогами                     |

`engine-merged-pr5` — имя checkpoint того же состояния, а не самостоятельная
ветка разработки. Три engine-имени нельзя объединять между собой: сейчас это
три ссылки на один commit, а не три набора изменений.

## Активные ветки курса

| Ветка | Ответственность                                                                                            |
| ----- | ---------------------------------------------------------------------------------------------------------- |
| `Hw5` | тесты, regression coverage и локальная debug-зона; `debug` сам по себе не называется автоматическим тестом |
| `Hw6` | SAST внешним инструментом, исправление и документирование найденных проблем                                |
| `Hw7` | SCA, SBOM, разбор зависимостей и исправление простых уязвимостей                                           |

`Hw6` строится поверх итоговой `Hw5`, а `Hw7` — поверх итоговой `Hw6`.
После переписывания базы их нужно rebase-ить в таком порядке.

## Не заглушки

- `Hw8` уже содержит рабочие audit/OCR-коммиты. Несмотря на имя следующего
  задания, это не пустая точка старта и не должна описываться как заглушка.
- `docs/hw5-report-edit` — временная рабочая ветка документации, а не
  продуктовая линия развития.
- `docs/hw5-rfc-zoo`, `archive/*`, `benchmark/*` и ветки с суффиксом
  `*-mixed` сохраняют историю планирования, измерений или промежуточной
  очистки. Они не являются merge-целями.

## Правило продолжения

Перед началом новой работы нужно:

1. проверить уникальные коммиты ветки относительно выбранной базы;
2. для настоящей заглушки сначала синхронизировать базу;
3. держать feature, тесты/измерения и документацию в минимальном числе
   откатываемых коммитов;
4. не переносить промежуточные benchmark/debug-артефакты как продуктовый код.
