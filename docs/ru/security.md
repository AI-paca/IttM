# Политика безопасности

[English](../en/security.md) | [Документация](./README.md)

## Границы доверия

| Режим                     | Куда попадает документ                         | Уровень доверия              |
| ------------------------- | ---------------------------------------------- | ---------------------------- |
| Browser OCR               | Только память вкладки и browser workers        | Локальное устройство         |
| Local Tesseract / EasyOCR | Nginx, gateway и Python OCR на локальном хосте | Доверенный локальный runtime |
| External LLM              | Gemini, OpenRouter или другой выбранный API    | Сторонний провайдер          |

## Локальная обработка

- В Docker наружу публикуется только nginx. Gateway и OCR находятся во
  внутренней сети Compose.
- Локальные режимы отправляют исходный `File` как multipart. Frontend не
  переводит его в Base64 и не вызывает `arrayBuffer()` перед отправкой.
- Gateway проксирует тело запроса потоком и не сохраняет документ в базе данных.
- Python читает upload чанками, но перед OCR объединяет его в полный объект
  `bytes`.
- Обычные изображения обрабатываются из памяти. PDF временно записывается в
  каталог `tempfile`, потому что Poppler принимает путь к файлу; каталог
  удаляется после запроса.

Проект не заявляет абсолютный zero-retention для операционной системы:
временные файлы PDF, swap, crash dumps и логи хоста зависят от окружения.

## Browser OCR

При выборе Browser исходный документ не отправляется в backend. Tesseract.js,
PDF.js и preprocessing выполняются в browser workers, где это поддерживается.
Весь PDF сначала читается в worker `ArrayBuffer`; для очень больших файлов это
остаётся риском памяти.

## Внешние LLM

- Gemini/OpenRouter требуют явного согласия в текущей сессии.
- Документ может быть уменьшен, закодирован в Base64 и отправлен провайдеру.
- Политика хранения и обработки после отправки определяется провайдером.
- API-ключ хранится в состоянии frontend и не передаётся локальному OCR backend.

Перед обработкой конфиденциальных документов необходимо проверить условия
выбранного провайдера и настройки браузера/хоста.

## Не закрытые риски

- Нет аутентификации для локального API: защита опирается на loopback bind и
  сетевую конфигурацию.
- Нет task ID, durable queue, retention policy и серверной отмены OCR.
- Upload целиком существует в памяти Python.
- Нет встроенной защиты от всех вариантов decompression/image bomb.
- NDJSON-ошибка после начала ответа возвращается событием `error` внутри HTTP
  200, а не новым HTTP status.
